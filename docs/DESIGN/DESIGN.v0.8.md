# 間取りプランナー Web アプリ 設計書

> Version: **0.8** (Draft)
> Last Updated: 2026-05-12
> Status: 実装中 (Phase 3 後半 UX 補強 v4)
>
> ## v0.7 → v0.8 の主な変更
>
> ### 1. 3D ドアパネルを openings 重なりに頑健化 (M56)
>
> v0.6 で `splitWallByOpenings` に `doorPanels[]` を追加して「壁の開口に必ず木調パネルを 1 枚」
> を実現したが、パネル push が `if (opWidth > 0)` の中にあった。複数の開口が同じ壁上で
> 重なる場合 (例: §6.2 の自動配置ドア + ユーザー追加ドア、密接した 2 つのドアなど)、
> 後続の opening は `opWidth = 0` (cursor が rightEdge 以降) で扱われ、**ドアパネルも描画から外れて
> しまう** 事故が発生していた。
>
> 修正方針 (M56):
>  - **doorPanels の push を cursor ベースの clip ロジックから分離する** — `op.kind === 'door'` の
>    開口部については毎ループ必ず判定し、ワールド/ローカルから直接 `panelLeft/panelRight` を
>    計算 (`Math.max(-total/2, center-half)` 〜 `Math.min(total/2, center+half)`) → 50mm 以上の
>    幅があれば確実にパネルを push
>  - 壁の solids / sill / lintel は引き続き cursor ベースで進める (壁内の "穴" のレイアウト用)
>  - 視覚的にはドアパネルが必ず 1 枚は出るので、`2D で配置したドアが 3D に出ない` 事故が
>    全て解消する
>
> ### 2. 窓のヒット領域を改善 (M57)
>
> `WindowMark` は壁の両側に平行な 2 本の線を描く図記号で、ストローク幅が 1〜2px のため
> 線そのものをピクセル精度で当てないとクリックが効かなかった。
>
> 修正: 窓の開口幅 × 壁厚 (= 二本線の間隔程度) を覆う **透明な Rect** を `Group` に追加し、
> クリック/タップのヒット対象にする。視覚は変わらないがクリック判定が広い。
>
> ### 3. ホイールズームの感度向上 (M58)
>
> `handleWheel` の指数係数 `0.001` ではホイール 1 ノッチで約 10% しか拡縮しなかった。
> 体感「もっと速く拡大したい」というユーザー要求に応じ、係数を **0.003** へ。
> ホイール 1 ノッチで約 26% の拡縮 (10% → 26%、約 2.6 倍速)。
>
> 副作用:
>  - 既存 M33 の「カーソル位置基準で pan を補正」ロジックはそのまま動作
>  - クランプ範囲 [0.02, 0.5] は変更なし
>
> v0.7 以前の §1〜§17 はそのまま継承。
